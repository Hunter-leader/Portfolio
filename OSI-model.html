OSI Model & Layer

December 17, 2024

The OSI (Open Systems Interconnection) Model is a conceptual framework used to understand and implement network communications. It divides communication into seven layers, each with a specific function



OSI MODEL

The full form of OSI is Open Systems Interconnection. In a network, when communication occurs between one device and another, it passes through each layer of the OSI model. The OSI model has a total of seven (7) layers, which are divided into three (3) categories.

The top three layers are known as Software Layers, the bottom three layers are called Hardware Layers, and the middle layer is referred to as the Core Layer.

All the applications we use in the network operate in the top three layers (Application Layers), while the hardware (such as Routers, Switches, Firewalls) is used in the bottom three layers (Hardware Layers). The Transport Layer interfaces between the Software Layers and Hardware Layers, which is why it is called the Core Layer.

Application Layer:

This is the seventh layer of the OSI model. The Application Layer provides a user interface and processes network data. The tasks performed by the Application Layer include resource sharing, remote file access, directory services, etc.

Presentation Layer:

The Presentation Layer acts as a data translator for network services. The tasks performed by this layer include data conversion, data compression, and decryption.


The common data formats used in this layer are:

  • .jpg
  • .mpeg

This layer ensures that data sent from the sender is readable and usable by the receiver by converting it into an appropriate format.



Session Layer:

The Session Layer establishes, manages, and terminates the connection between the source and destination devices. It contruls the session and disconnects it when necessary.


For data transmission, three types of contrul are used:

  • simplex:
    • In simplex mode, data flows only in one direction.
    • Example:A keyboard sending input to a monitor.

  • Half-Duplex:
    • In half-duplex mode, data flows in both directions, but one direction at a time.
    • Example:Walkie-talkie communication.

  • Full-Duplex:
    • In full-duplex mode, data flows simultaneously in both directions.
    • Example:Telephone conversations.

Transport Layer:

The fourth layer of the OSI model is the Transport Layer. The function of this layer is to ensure that the data received from the Session Layer is reliably delivered to the other device.

In the Transport Layer, two types of transmission are used to deliver the data:

  • Connection-Oriented:
    • Before sending data, the sender establishes a connection with the receiver through an Acknowledgment Signal.
    • This is a reliable method where the delivery of data is guaranteed.
    • Example: TCP (Transmission Contrul Protocul).


    • Connectionless:
      • Before sending data, the sender does not establish any Acknowledgment Signal with the receiver.
      • This is an unreliable method where the delivery of data is not guaranteed.
      • Example: UDP (User Datagram Protocul).

    Network Layer:

    The function of the Network Layer is addressing and packet delivery. In this layer, data is encapsulated by adding a network address to the data packet. Routers are used at this layer, and they create routing tables for directing the data packets to their destination.


    Data Link Layer:

    The second layer of the OSI model is the Data Link Layer. The Data Link Layer works similarly to the Network Layer, but with a key difference. While the Network Layer transfers data outside the same network, the Data Link Layer transfers data packets within the same network. In other words, if two devices are within the same network, the Data Link Layer enables the transmission of data packets between them. The Data Link Layer takes data packets from the Network Layer, divides them into smaller parts called frames, and then transmits them. Like the Network Layer, the Data Link Layer also handles data flow, data contrul, error contrul, etc. You can think of this layer as creating a logical link between two devices.


    Physical Layer:

    The Physical Layer is the first layer of the OSI Model, located at the bottom. From its name, you can guess the type of work this layer does. It establishes a physical connection between two devices and transmits data from one device to another using physical media, such as cables. This layer converts data into a bitstream, meaning it transforms the data into 0s and 1s. The data is transmitted bit by bit.
